### Product Requirements Document (PRD) v2.0

# Grammar Police — AI Personal Grammar Coach for Real Conversations

Versi: 2.0

Tujuan dokumen: menjadi spesifikasi implementasi untuk agent coding dalam mengembangkan versi produksi Grammar Police yang benar-benar adaptif sejak onboarding pertama pengguna.

### Ringkasan produk

Grammar Police adalah platform pembelajaran bahasa Inggris berbasis AI dengan konsep serious game bertema akademi kepolisian. Pengguna menjalani Police Academy Assessment melalui percakapan AI untuk memetakan kemampuan grammar awal. Hasil assessment menghasilkan Grammar Passport dan Personalized Learning Journey yang menentukan district, gate awal, tingkat kesulitan, serta fokus remedial pengguna.

Perbedaan utama dari versi sebelumnya adalah adaptive learning dimulai sejak onboarding, bukan setelah beberapa gate pertama.

### Tujuan produk

* Menyediakan latihan grammar kontekstual berbasis percakapan AI.

* Mendeteksi kekuatan dan kelemahan grammar pengguna sejak sesi pertama.

* Menyusun jalur belajar personal yang sesuai dengan hasil assessment.

* Meningkatkan engagement melalui gamifikasi, district progression, badge, dan rank progression.

* Menyediakan data analitik yang dapat digunakan untuk evaluasi pembelajaran dan pengembangan fitur adaptif.

### Modul inti

### Choose Your Mission

* Menampilkan pilihan tujuan belajar: Academic, Job Interview, Traveling, Daily Conversation, dan Business English.

* Pilihan disimpan sebagai `learning_goal` pada profil pengguna.

* Tujuan belajar digunakan sebagai konteks awal untuk assessment dan pemilihan district.

### Police Academy Assessment

* Berupa percakapan AI selama 5–10 menit.

* Pertanyaan harus menguji kategori grammar utama: present tense, past tense, future tense, present perfect, conditional, question forms, passive voice, dan articles.

* Contoh pertanyaan: “Tell me about your daily routine.”, “What did you do last weekend?”, “What are you going to do tomorrow?”, “Have you ever traveled abroad?”, dan “If you had one million dollars, what would you do?”

* Jawaban pengguna dievaluasi oleh LLM dan dikonversi menjadi skor per kategori grammar.

### AI Analysis

* Menampilkan proses analisis jawaban pengguna.

* Mengidentifikasi kekuatan grammar, kelemahan grammar, dan pola kesalahan awal.

* Menentukan `overall_mastery` dan `initial_rank` pengguna.

### Grammar Passport

* Menampilkan persentase penguasaan grammar secara keseluruhan.

* Menampilkan skor per kategori: Present Tense, Past Tense, Future Tense, Passive Voice, Question Forms, dan Articles.

* Menjadi profil kemampuan yang diperbarui setelah setiap mission atau remedial selesai.

### Personalized Learning Journey

* AI memilih district awal berdasarkan hasil assessment dan tujuan belajar pengguna.

* District utama: Travel District, Student District, Workplace District, Presentation District, dan Interview District.

* Setiap district berisi gate challenge yang menyesuaikan tingkat kesulitan dengan profil pengguna.

| Remedial system     | Menyediakan latihan tambahan jika jawaban pengguna belum tepat.                          |
| Progress system     | Menghitung XP, coins, badge, rank, dan district completion.                              |

### Adaptive learning logic

| Kondisi hasil assessment  | Perilaku sistem                                                                    |
| ------------------------- | ---------------------------------------------------------------------------------- |
| Skor Present Tense ≥ 90%  | Gate present dapat dipersingkat dan user langsung melanjutkan ke topik berikutnya. |
| Skor Present Tense 50–70% | User memulai dari Present Gate standar.                                            |
| Skor Present Tense < 50%  | User masuk ke Present Gate dengan remedial tambahan sebelum melanjutkan.           |
| Passive Voice rendah      | AI menambahkan Passive Voice sebagai prioritas pada district berikutnya.           |
| Question Forms rendah     | AI menambahkan latihan question forms pada gate awal.                              |

### Gamification system

* Rank: Grammar Cadet sebelum assessment; setelah assessment AI menentukan rank awal seperti Cadet I, Cadet II, atau Explorer.

* XP: diberikan untuk menyelesaikan gate challenge dan remedial.

* Coins: diberikan sebagai reward dan dapat digunakan untuk fitur kosmetik atau bantuan belajar.

* Badges: diberikan untuk pencapaian tertentu, misalnya Present Tense Explorer.

* District completion: pengguna harus menyelesaikan district sebelum membuka district berikutnya.

* Daily mission: memberikan fokus latihan harian berdasarkan kelemahan terbaru pengguna.

### Frontend requirements

* Menggunakan Next.js + React untuk frontend.

* Menggunakan Tailwind CSS dan mempertahankan design system dari prototype `grammar-police.html`.

* Seluruh screen pada storyboard harus dibuat sebagai komponen React terpisah.

* State management dapat menggunakan Zustand atau Context API.

* UI harus responsif untuk desktop dan mobile.

* Animasi loading, progress bar, badge, dan rank progression harus tetap ringan dan tidak mengganggu performa.

### Backend requirements

* Menggunakan Node.js + Express atau FastAPI.

* Menggunakan MySQL sebagai database utama.

* Mengintegrasikan LLM provider seperti OpenAI GPT-4o atau Gemini 2.5 Flash.

* Semua endpoint harus menggunakan autentikasi JWT.

* Hasil evaluasi AI harus disimpan untuk analisis pola kesalahan dan pembaruan Grammar Passport.

### Non-functional requirements

* Performance: waktu respons API untuk evaluasi AI maksimal 5 detik.

* Reliability: sistem harus memiliki fallback jika LLM gagal merespons.

* Scalability: arsitektur harus mendukung penambahan district dan kategori grammar baru.

* Security: input pengguna harus divalidasi dan disanitasi sebelum dikirim ke LLM.

* Accessibility: aplikasi harus mendukung preferensi `prefers-reduced-motion` dan memiliki kontras warna yang memadai.

### Definition of done

* Pengguna dapat memilih tujuan belajar dan menyelesaikan Police Academy Assessment.

* AI menghasilkan Grammar Passport dengan skor per kategori grammar.

* Sistem membuat Personalized Learning Journey berdasarkan hasil assessment.

* Gate challenge memberikan evaluasi AI dan remedial jika diperlukan.

* Progress pengguna tersimpan dan memperbarui Grammar Passport secara berkala.

* Seluruh screen pada storyboard 20 langkah dapat diakses dan berfungsi sesuai alur.

### Instruksi untuk agent coding

* Implementasikan Police Academy Assessment sebagai modul onboarding baru sebelum Start Journey.

* Jangan gunakan evaluasi regex statis; gunakan LLM dengan output JSON terstruktur.

* Pastikan hasil assessment menjadi input utama untuk adaptive learning engine.

* Buat komponen frontend sesuai urutan storyboard 20 langkah.

* Gunakan design token dan gaya visual dari prototype yang sudah ada agar tampilan tetap konsisten.

* Fokus implementasi awal pada flow: authentication → mission selection → assessment → passport → personalized journey → gate challenge.

PRD ini dapat langsung digunakan sebagai panduan implementasi untuk agent coding dalam mengembangkan versi baru Grammar Police yang lebih adaptif, imersif, dan sesuai dengan storyboard terbaru.
